Идея:

1) Указываем пары строк с толкованием и пометами (от диеза "#" - начало строки толкования, до начала примера употребления слова "{{пример|").

2) Берём только такие пары слов, для которых явно указана связь в одной из статей. Например, в статье "полотно" указан синоним "ткань". 
Эта связка "полотно - синоним - ткань" тоже должна указываться во входных данных теста.

3) Нужно указать примеры для разных типов семантических отношений. Только для русских слов.


полотно - ткань

полотно

# {{текст.}} гладкая и плотная ткань из однородных нитей (чаще всего льняных) простейшего переплетения, выработанная из основы и [[уток|уткa]] одинаковой толщины и плотности; в широком смысле — текстильное изделие плоской формы, изготовленное тканым или нетканым способом, а также плоские изделия из других материалов

ткань

# {{текст.|ru}} тканая [[материя]], [[материал]]

Задача:
1) извлечь данные из XML-файла, представить в виде структуры в PHP.
2) Проверить (сравнить с данными базы данных wikt_parsed):

- для каждого SourceWord убедиться, что для слова <word> есть в точности такой же <meaning>.
- для каждой пары слов <TwoWords> есть указанное семантическое отношение <SemanticRelation>,
причём это отношение может быть указано как в статье о первом слове, так и во втором, или сразу в обоих.

3) Если проверка даёт отрицательный результат, то вывести номер строки XML-файла, 
который нужно вручную проверить и сравнить с современными данными Викисловаря.
+ указать, в чём именно ошибка - нет такого отношения или такого толкования.
